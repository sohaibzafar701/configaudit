What is NCRT?

NCRT (Network Configuration Rule Tester) is a powerful security auditing tool designed to analyze network device configurations and identify security vulnerabilities and compliance issues.

Key Capabilities:

  • Analyze router, switch, and firewall configurations
  • Identify security vulnerabilities and misconfigurations
  • Check compliance with security frameworks
  • Track devices and audit history
  • Generate detailed reports with actionable findings

Supported Vendors

  • Cisco (IOS, IOS-XE, NX-OS)
  • Juniper (JunOS)
  • Extensible architecture for more vendors

Security Focus

  • Authentication & Authorization
  • Encryption & Cryptography
  • Access Control
  • Network Services
  • Compliance Frameworks

Getting Started

1

Upload Your Configuration

Navigate to the Audit page from the sidebar. You can either:

  • Drag and drop your configuration file (.txt, .cfg, .conf)
  • Click to browse and select a file from your computer
  • Paste configuration content directly (if supported)
2

Enter Device Information

Provide the following information:

  • Device Identifier (required): IP address, hostname, or device name
  • Device Make (optional): e.g., Cisco, Juniper
  • Device Model (optional): e.g., Catalyst 9300
  • Device Type (optional): Router, Switch, or Firewall

Tip: The system will attempt to auto-detect device information from the configuration if not provided.

3

Select Rule Tags

Choose which security rules to apply by selecting relevant tags:

  • Vendor tags: cisco, juniper (vendor-specific rules)
  • Category tags: authentication, encryption, access-control
  • Generic tags: generic, all (applies to all devices)

Tip: Select multiple tags to run a comprehensive audit. The system will only run rules that match your device type.

4

Start the Audit

Click the "Start Audit" button to begin the analysis. The system will:

  • Parse your configuration file
  • Extract device metadata (hostname, model, firmware, etc.)
  • Run selected security rules against the configuration
  • Generate findings with severity levels and recommendations

Note: Audit processing happens in the background. You can monitor progress in real-time.

5

Review Results

Once the audit completes, navigate to the Report page to:

  • View detailed findings organized by severity and category
  • Filter and search through results
  • Export reports in multiple formats (PDF, CSV, HTML, JSON)
  • Review compliance scores and statistics

Detailed Tutorials

Audit Page

The Audit page is where you start new security audits of your network device configurations.

Uploading Configurations

  • Supported file formats: .txt, .cfg, .conf
  • Maximum file size: 10MB (configurations are typically much smaller)
  • You can upload the raw configuration output from your device
  • The system supports both full configurations and partial snippets

Device Identifier

The device identifier is used to track and organize your audits. Use a unique identifier such as:

  • IP address (e.g., 192.168.1.1)
  • Hostname (e.g., router-01.company.com)
  • Device name (e.g., Core-Switch-01)

Rule Tag Selection

Rule tags help you focus the audit on specific areas:

  • Vendor-specific tags: Select tags matching your device vendor for vendor-specific checks
  • Category tags: Choose security categories you want to audit (authentication, encryption, etc.)
  • Generic tags: Apply to all device types for general security checks

Only enabled rules matching your selected tags and device type will be executed.

Monitoring Progress

After starting an audit, you'll see a progress section showing:

  • Overall progress percentage
  • Current rule being executed
  • Number of rules completed
  • Detailed execution information for each rule

Report Page

The Report page displays all your audit results with powerful filtering and export capabilities.

Selecting an Audit

Use the audit dropdown at the top to select which audit results you want to view. The list shows:

  • Device identifier and hostname
  • Audit date and time
  • Status (Completed, Failed, Processing)
  • Total findings count

Filtering Results

Use the filter options to narrow down findings:

  • Severity: Filter by Critical, High, Medium, Low, or Info
  • Category: Filter by security category (authentication, encryption, etc.)
  • Search: Search for specific text in findings
  • Status Filter: Show only completed, failed, or processing audits
  • Date Range: Filter audits by creation date

Viewing Findings

Findings are displayed with:

  • Severity badges (color-coded)
  • Rule name and description
  • Detailed finding messages
  • Configuration context (where the issue was found)
  • Recommendations for remediation

Exporting Reports

Export your audit results in multiple formats:

  • PDF: Professional formatted report for sharing
  • CSV: Spreadsheet format for data analysis
  • HTML: Web-friendly format
  • JSON: Machine-readable format for integration

Rules Page

The Rules page allows you to view and manage the security rules used in audits.

Viewing Rules

The Rules page displays all available security rules with:

  • Rule name and description
  • Severity level (Critical, High, Medium, Low, Info)
  • Category (authentication, encryption, etc.)
  • Rule type (pattern, python, hybrid)
  • Applicable tags (vendor, category)
  • Enabled/disabled status

Filtering Rules

Use the filter options to find specific rules:

  • Vendor: Filter by device vendor (Cisco, Juniper, etc.)
  • Category: Filter by security category
  • Severity: Filter by severity level
  • Type: Filter by rule type
  • Search: Search by rule name or description

Rule Details

Click on a rule to view detailed information including:

  • Full rule description
  • YAML rule definition
  • Compliance framework mappings
  • Rule execution history

Assets Page

The Assets page helps you track and manage all your network devices and their audit history.

Asset Overview

Each asset (device) shows:

  • Device identifier and hostname
  • Device make, model, and type
  • Last audit date and status
  • Total number of audits performed
  • Latest findings count

Audit History

Click on a device to view its complete audit history:

  • All audits performed on the device
  • Audit dates and statuses
  • Finding counts per audit
  • Links to detailed reports

Search and Filter

Use the search and filter options to:

  • Search by device identifier, hostname, model, or make
  • Filter by last audit date (last 7/30 days, never audited)
  • Quickly find devices that need attention

Analyze configurations from multiple network device vendors including Cisco, Juniper, and more. The extensible architecture allows for easy addition of new vendor parsers.

Flexible Rule System

Compliance Tracking

Track compliance with security frameworks and requirements. View compliance scores and identify gaps in your security configuration.

Multiple Export Formats

Export audit reports in PDF, CSV, HTML, and JSON formats. Share results with stakeholders or integrate with other security tools.

Audit History & Tracking

Maintain a complete history of all audits performed. Track changes over time, compare configurations, and monitor security improvements.

Understanding Audit Results

Severity Levels

Critical

Immediate security risks that require urgent attention. These findings indicate severe vulnerabilities that could lead to unauthorized access or data breaches.

High

Significant security issues that should be addressed promptly. These findings represent substantial risks to network security.

Medium

Moderate security concerns that should be reviewed and addressed. These findings indicate potential security weaknesses.

Low

Minor security observations or best practice recommendations. These findings are informational but worth reviewing.

Info

Informational findings that provide context or additional information about the configuration. No immediate action required.

Finding Structure

Findings are organized in a parent-child structure:

  • Parent Findings: Represent the overall rule result. The parent severity is set to the highest severity among its children.
  • Child Findings: Represent specific instances where the rule found issues. Multiple child findings can be grouped under a single parent.

Tip: When a rule finds multiple issues, they're grouped together under one parent finding for easier review.

Compliance Scores

Compliance scores provide an overall assessment of how well your configuration aligns with security frameworks:

  • Scores are calculated based on the findings and their severity
  • Higher scores indicate better compliance
  • Scores are broken down by framework and requirement level
  • Use scores to track improvements over time

How to Act on Findings

  1. Prioritize by Severity: Start with Critical and High severity findings first.
  2. Review Recommendations: Each finding includes recommendations for remediation.
  3. Check Configuration Context: Review the specific configuration lines where issues were found.
  4. Make Changes: Update your device configurations based on the findings.
  5. Re-audit: Run a new audit after making changes to verify improvements.
  6. Track Progress: Use the Assets page to track audit history and monitor improvements over time.
